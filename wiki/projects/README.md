# wiki/projects/

Project-specific pages — one page per active or archived project.

A project page typically has:

- Goal and current status
- Key decisions and open questions
- Links to relevant sources in `raw/`, concepts, and people
- A short timeline / changelog

Filenames: kebab-case (`dissertation-proposal.md`, `vault-setup.md`). When a project is archived, prefix it with `archive-` or move it to a future `projects/archive/` subfolder.
